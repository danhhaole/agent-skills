# Deep Reader Skill

**Language / Ngôn ngữ / 语言:** [English](./README.md) | [Tiếng Việt](./README.vi.md) | [中文](./README.zh.md)

这个技能帮助智能体深度阅读长篇书籍和论文——采用一位认真的人类研究者会用的方式，而不是把整段文本硬塞进一个 prompt。

## 这里所说的"深度阅读"是什么意思

LLM 的注意力会随着上下文变长而逐渐分散，容易忽略埋在长 prompt 中间的内容（即"lost in the middle"现象）。把一本 500 页的书一次性塞进上下文再提问，得到的回答会在讨论到第 12 章时，悄悄忘记第 3 章说过什么。

这个技能用认真的人类研究者一直在用的方法来解决这个问题：不要一次性把整本书都放进工作记忆里。它把三种成熟的阅读方法组合成一条流水线：

- **Mortimer Adler 在《如何阅读一本书》中提出的方法** —— 检视阅读（inspectional）、分析阅读（analytical）、主题阅读（syntopical），按此顺序依次应用。
- **SQ3R 的 Recite 步骤** —— 草拟完一章的笔记后，再把这一章浏览一遍，逐条对照原文核实论点和引文，然后才进入下一章。
- **S. Keshav 的三遍阅读法**，用于论文、学位论文和综述。

这本书按遍数分层阅读——先通读一遍把握结构，再逐章深入阅读内容——在阅读过程中，把学到的一切都外化写入一个**按页码锚定的笔记工作区**。笔记成为持久记忆；原文只在正在精读某一章或需要核实某个论点时才需要重新调出。页码是坐标系统，让每一条论点都能追溯回原文出处，无论是在当前会话还是之后的会话中。

## 两种模式

- **overview（概览模式）** —— 只执行 Adler 的检视阅读：一份全书结构地图，加上围绕你阅读目的、聚焦最重要章节的定向摘要。速度快，不写逐章笔记。
- **study（精读模式）** —— 完整流水线：确定阅读目的 → 检视阅读地图 → 逐章分析笔记并配合 Recite 核实 → 回答 Adler 四个问题的层级式综合 → 机械化引文核验。

由你来选择模式（或者智能体会根据书的篇幅和你说明的阅读目的提出建议，再由你确认或调整）。

## 什么时候使用这个技能

当你要求智能体阅读、研究、总结、分析、审阅，或回答关于一本长篇书籍、教材、PDF、EPUB、学位论文、综述论文，或任何大约 50 页以上文档的问题时，都可以使用它。它同样适用于：如果某本书上一次会话留下的笔记工作区已经存在于源文件旁边，而你又提出了关于这本书的后续问题——这时智能体会先在已有笔记中查找，而不是从头重新阅读。

不要指望它在处理短文档（大约 30 页以下）时也会触发：这类文档完全可以放进上下文里直接读，整套流水线只会是多余的开销。

**触发短语：** "帮我读一下这本书"、"研究一下这份 PDF"、"总结这本教材"、"分析这篇论文"、"深度阅读这篇学位论文"、"tóm tắt sách"、"phân tích luận án"、"nghiên cứu paper này"

## 依赖要求

- **bash** —— 必需，所有脚本都是 POSIX 风格的 shell 脚本（依赖 coreutils、awk、grep、sed）。
- **pdftotext**（来自 **poppler**）—— 仅当源文件是 PDF 时需要。
- **pandoc** —— 仅当源文件是 EPUB/DOCX 时需要。TXT/MD 源文件两者都不需要。

macOS 安装：

```bash
brew install poppler pandoc
```

Debian/Ubuntu 安装：

```bash
apt-get install poppler-utils pandoc
```

如果缺少依赖，准备文本的脚本会给出清晰的错误信息并退出，而不是静默失败；此时技能会回退为直接用智能体的 Read 工具按页批量读取 PDF/EPUB。走这条回退路径时，你会失去机械化的引文核验步骤，因此智能体会通过更频繁地手动重新核对每一处引用来弥补。

## 使用示例

在支持 skills 的智能体中，可以这样提出任务：

```text
深度阅读《领域驱动设计》（Domain-Driven Design），告诉我它对我们目前服务边界划分
有什么启发——我要完整的 study 模式，不只是摘要。
```

```text
明天开会前我只需要大致了解这本 400 页的手册——给我一个 overview：主要分成哪几
部分，哪些章节对正在搭建 BI 数据管道的人真正重要？
```

```text
（同一本书，后续提问）书里关于聚合根（aggregate root）和限界上下文
（bounded context）是怎么说的？先查一下你已有的笔记。
```

对第三个例子，智能体会去查已经建好的工作区中的 `terms.md` 和 `notes/` 文件，而不是重新读一遍这本书——这正是边读边把笔记外化的全部意义所在。

## 笔记工作区

流水线产生的一切都保存在一个工作区目录中，默认位置是源文件的同级目录，这样它才能跨会话保留下来：

```text
<slug>-notes/
├── source.txt          # 带有 [[page N]] 标记的全文——坐标系统
├── chapters.tsv         # 已确认的结构：chNN<TAB>from<TAB>to<TAB>title
├── chapters/            # 由 split-chapters.sh 切分出的各章文件
│   └── ch01-<slug>.txt
├── map.md               # 检视阅读阶段的产出
├── terms.md             # 跨章节的关键术语台账
├── notes/
│   └── ch01-<slug>.md   # 每章一份分析笔记
└── synthesis.md         # 最终综合结论 + 核验日志
```

页码把每一条笔记都锚定回 `source.txt`，这正是后续会话能够通过查笔记而不是重读全书来回答后续问题的关键所在。

## 脚本一览

七个脚本负责流水线中机械化的部分；围绕它们的判断工作（确认结构、撰写笔记、综合总结）则由智能体完成。

| 脚本 | 作用 |
| --- | --- |
| `prepare-text.sh` | 把源文件转换成带页码的 `source.txt`，创建工作区（幂等） |
| `extract-structure.sh` | 启发式的结构初稿——只是一个初步猜测，需要与真实目录核对 |
| `read-pages.sh` | 打印某一页或某个页码范围，保留 `[[page N]]` 标记 |
| `search-book.sh` | 在全书范围内 grep 搜索，结果标注页码 |
| `split-chapters.sh` | 把已确认的章节切分成独立文件 |
| `build-diagram.sh` | 根据已确认结构机械生成 Mermaid 思维导图 |
| `verify-quotes.sh` | 揪出编造的引文和错误的页码引用 |

第八个脚本 `self-test.sh` 不属于阅读流水线本身——它是一个"安装体检"工具。每当脚本表现异常时运行 `bash ./scripts/self-test.sh`：它会自己生成一份合成测试数据，把每个脚本都实际跑一遍，并报告可选依赖（`pdftotext`、`pandoc`）是否存在，帮你判断问题出在环境上还是用法上。

## 从设计上防止编造内容

study 模式笔记中的每一条引文都会经过机械化核验：`verify-quotes.sh` 会对引文字符串做归一化处理，再与所引页码进行比对（允许 ±1 页的误差，以覆盖句子跨页的情况），并把任何编造或页码引用错误的内容标记为 `FAIL`/`NEAR`。在实际测试中，这条流水线曾被用来深度阅读三本 370–600 页的真实书籍——《领域驱动设计》（Domain-Driven Design）、《启示录》（Inspired），以及一本 ERP/BI 手册——在其中两次 study 模式运行中，核验器分别以 458/458 和 120/120 条引文全部通过。

## 安装

### 1. 使用 CLI（推荐）

```bash
npx skills add tronghieu/agent-skills --skill deep-reader
```

### 2. 手动安装（适合非技术用户）

1. **下载：** 转到此仓库的 `skills/` 文件夹下载 `deep-reader.zip` 文件。
2. **解压和复制：** 解压 `deep-reader.zip` 并将 `deep-reader` 文件夹复制到以下目录之一：

**针对特定项目：**
将 `deep-reader` 文件夹复制到项目根目录下的 `.agents/skills/` 或 `.claude/skills/`。

**全局安装（所有项目可用）：**
* **Mac / Linux：** `~/.agents/skills/` 或 `~/.claude/skills/`
* **Windows：** `%USERPROFILE%\.agents\skills\` 或 `%USERPROFILE%\.claude\skills\`（通常为 `C:\Users\<YourUsername>`）
