# Scene List — `scene-list.md`

Phase 4, led by Scheherazade. Breaks the outline into the actual scenes to be
drafted. This is the bridge from plan to prose — each row becomes a drafting
target in Phase 5. Keep it a working table, not a novel; one tight entry per scene.

## Per-scene entry

For each scene, capture:

- **#** — scene/chapter number
- **POV** — whose viewpoint (and tense, if it varies)
- **Setting** — where/when (cross-check `world-bible.md`)
- **Characters** — who's present
- **Goal** — what the POV character wants in this scene
- **Conflict** — what opposes it
- **Turn** — the value that changes (e.g. safe→threatened); if nothing turns,
  cut or merge the scene
- **Outcome & hook** — how it ends and what pulls the reader onward
- **Purpose** — why this scene exists in the book (plot? character? both?)

## Format

A markdown table or a numbered list of compact entries. Example entry:

```
### Scene 3 — Ch.2
- POV: Mara (past tense)
- Setting: the flooded archive, midnight
- Characters: Mara, the Archivist
- Goal: Mara wants the sealed ledger
- Conflict: the Archivist won't break his oath
- Turn: trust → suspicion
- Outcome & hook: he gives a forgery; she doesn't yet know
- Purpose: plants the forgery payoff (Ch.9); deepens Mara's paranoia
```

> A scene with no goal/conflict/turn is a candidate for cutting. Flag those for
> Aristotle (`scene audit`) rather than drafting dead weight.
